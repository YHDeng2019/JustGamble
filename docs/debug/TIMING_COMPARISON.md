# 单人模式 vs 联机模式时间配置对比

## 单人模式 (Game.jsx) 的时间配置

### 发牌动画
- 初始延迟：500ms (initGame 后等待)
- 每张牌间隔：150ms
- 发牌完成后延迟：400ms
- 人类玩家首先行动时额外延迟：1500ms

### 回合结束延迟
- **摊牌结束 (processTurn)**：1500ms
- **摊牌结束 (handleAction)**：2000ms
- **弃牌结束**：500ms
- **只剩一人**：500ms

### Toast 显示时间
- 摊牌后停留：6500ms
- 弃牌结束停留：3500ms

### 阶段切换
- 阶段切换停顿：800ms (让玩家看清公共牌)

### ActionBar 显示延迟
- 延迟显示：600ms (避免连续决策无间隔)

### All-in 戏剧效果
- 持续时间：1400ms

### 新一局开始
- 桌面过渡动画：600ms
- dealInitialCards 延迟：600ms
- animateDealing 延迟：500ms

### AI 思考时间
```javascript
// 基础时间：700ms
// 跨度：1600ms × 纠结度 × 下注压力
// 随机量：400ms
const base = 700;
const span = 1600 * ambiguity * facingBet;
thinkTime = base + span + Math.random() * 400;
// 范围：700-2700ms
```

### 快速摊牌 (所有人 All-in)
- 每阶段间隔：400ms

## 联机模式需要同步的配置

### ❌ 缺失的配置
1. **回合结束延迟不一致**
   - 单人：摊牌 1500ms/2000ms，弃牌 500ms
   - 联机：无延迟直接处理
   
2. **阶段切换停顿**
   - 单人：800ms
   - 联机：无

3. **All-in 持续时间**
   - 单人：1400ms
   - 联机：2000ms（不一致）

4. **新手牌开始延迟**
   - 单人：600ms 桌面过渡 + 600ms dealInitialCards
   - 联机：3000ms 直接延迟（房主控制）

5. **快速摊牌逻辑**
   - 单人：有完整的 quickDealToShowdown (400ms/阶段)
   - 联机：无
